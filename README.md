# Local LLM Video RAG System

![Python](https://img.shields.io/badge/Python-3.x-blue)
![Ollama](https://img.shields.io/badge/Ollama-local-green)
![License](https://img.shields.io/badge/license-MIT-blue)

A fully local Retrieval-Augmented Generation (RAG) pipeline for semantic question answering over video content. Videos are transcribed, chunked, embedded, and queried entirely offline — no external APIs required.

---

## Tech Stack

| Tool | Purpose |
|---|---|
| FFmpeg | Video to MP3 conversion |
| Whisper | Speech-to-text transcription |
| nomic-embed (Ollama) | Embedding generation |
| Pandas + joblib | In-memory embedding storage |
| scikit-learn | Cosine similarity retrieval |
| Mistral (Ollama) | Answer generation |

---

## Pipeline

```
Video File
    │
    ▼
mp3_Conversion.py       →  Extracts audio as MP3 using FFmpeg
    │
    ▼
Speech_To_Text.py       →  Transcribes MP3 to text using Whisper
    │
    ▼
new_Chunks.py           →  Groups segments into chunks, generates embeddings via nomic-embed
    │
    ▼
readChunks.py           →  Loads embeddings into a Pandas DataFrame (persisted with joblib)
    │
    ▼
processing_Query.py     →  Accepts user query → embeds query → cosine similarity retrieval → Mistral generates answer
```

## How to Run

### Prerequisites

- Python 3.x
- [FFmpeg](https://ffmpeg.org/download.html) installed and added to PATH
- [Ollama](https://ollama.com) running locally with the following models pulled:

```bash
ollama pull mistral
ollama pull nomic-embed-text
```

### Install Dependencies

```bash
pip install -r requirements.txt
```

### Quick Start (Windows)

Place your video file in the project directory according to the expected folder structure (e.g. `1_lists in python`), then run:

```
run_pipeline.bat
```

### Manual Steps

Run scripts in the following order from the `pipeline/` directory:

```bash
python mp3_Conversion.py
python Speech_To_Text.py
python new_Chunks.py
python readChunks.py
python processing_Query.py
```

After `processing_Query.py` starts, enter your question in the CLI. The system retrieves the most relevant transcript chunks and generates a context-aware answer using Mistral.

---

## Chunking Strategy

Whisper's default transcript segments are often very short (sometimes single words), which degrades retrieval quality due to limited semantic context. To address this, consecutive segments are grouped into larger chunks (5 segments per chunk), improving contextual coherence and retrieval relevance.

This highlights a core RAG trade-off: finer chunks improve precision but lose context, while larger chunks retain context but may introduce noise.

---

## Retrieval Logic

Cosine similarity (via `sklearn.metrics.pairwise`) is used to rank transcript chunks against the query embedding. Since sklearn's implementation handles normalization internally, no external vector normalization is needed. The top-k most similar chunks are passed as context to Mistral for answer generation.

## Limitations

- Input file path is currently hardcoded — dynamic input is planned
- Embeddings are stored in-memory and are not suited for large-scale datasets
- No formal retrieval evaluation metric (e.g. Hit Rate, MRR) implemented
- Whisper transcription is computationally intensive on CPU
